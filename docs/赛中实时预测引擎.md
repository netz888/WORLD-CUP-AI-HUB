# 赛中实时预测引擎

<sub>[← 项目根](../README.md) · [数据拉取与实时机制](./数据拉取与实时机制.md) · [lib](../lib/README.md) · [components](../components/README.md)</sub>

> 在比赛进行中，结合**实时比分 + 剩余时间 + 红黄牌 + 实时技术统计（场面权重）**逐秒重算最终胜平负、最可能比分与后续进球概率。
> 与全站一致：**纯数学函数、可复现、零 AI 推理、零后端负载**。仅供娱乐，不构成投注建议。

---

## 1. 整体逻辑

```
赛前引擎 prediction-v2 (opus4.8)
   └─► 烘焙出每场期望进球 λ（match.detail.xgHome / xgAway）
                │
                ▼
赛中引擎 lib/in-play.ts  predictInPlay()        ← 纯函数，客户端运行
   ① 时间衰减   λ_rem = λ_pre × (90 − t) / 90
   ② 红/黄牌修正
   ③ 比分状态效应（领先收缩 / 落后压上）
   ④ 场面权重（实时技术统计：控球/射门/射正/xG/角球 → 占优方上调）
   └─► 对"剩余进球"跑 Dixon-Coles 联合分布
                │
   最终结果 = 当前真实比分 + 剩余推演
                ▼
组件 components/in-play-analysis.tsx           ← 常驻三态卡片
   未开赛(t=0) / 进行中(逐秒) / 已完赛(t=90 锁定)
   + SofaScore 式技术统计对比条（赛中实时、已完赛锁定、未开赛隐藏）
```

**核心思想**：不重新训练、不二次估计赛前实力，**直接复用赛前已校准的 λ**，只对"比赛还剩下的那部分时间"做概率推演，再把已经发生的真实比分叠加上去。随着比赛推进，剩余时间趋于 0，λ_rem → 0，结果自然向当前比分**收敛锁定**。

## 2. 引擎逻辑（`lib/in-play.ts`）

入口为纯函数 `predictInPlay(input, P = ACTIVE_PARAMS)`，无副作用、无 I/O，可在浏览器逐秒调用。

### 2.1 输入 / 输出

| 输入 `InPlayInput` | 含义 |
| --- | --- |
| `lambdaPreHome` / `lambdaPreAway` | 赛前模型期望进球（取自 `match.detail.xgHome/xgAway`） |
| `minute` | 当前比赛分钟（中场休息按 45，未开赛 0，已完赛 90） |
| `homeScore` / `awayScore` | 当前真实比分 |
| `redHome` / `redAway` | 红牌数（可选） |
| `yellowHome` / `yellowAway` | 黄牌数（可选） |
| `stats` | 双侧实时技术统计（可选，`{home, away}`，任一侧缺失则跳过场面权重）。每侧含 `possession`/`shotsTotal`/`shotsOnTarget`/`xg`/`corners`，均可空 |

| 输出 `InPlayOutput` | 含义 |
| --- | --- |
| `homeWin` / `draw` / `awayWin` | 最终结果胜平负（整数 %，强制和=100） |
| `confidence` | = 三者最大值 |
| `finalScore` / `topFinals` | 最可能最终比分 / Top4 最终比分 |
| `homeScoreMore` / `awayScoreMore` | 该队**后续还会再进球**的概率 |
| `lambdaRemHome` / `lambdaRemAway` | 剩余时间的期望进球（修正后） |
| `remPct` | 剩余时间占比 |
| `lean` | 当前倾向（`home`/`draw`/`away`） |
| `modifiers` | 生效的红黄牌数、比分效应（`stateEffect` / `stateIntensity`）与**场面权重**（`field`），供 UI 展示透明可解释 |
| `modifiers.field` | `{ active, homeShare, lean, intensity }`：是否生效、主队进攻压力占比(%)、偏向(`home`/`away`/`even`)、实际生效强度(%) |

### 2.2 四类修正（按计算顺序）

**① 时间衰减**

$$\text{rem} = \frac{90 - t}{90}, \qquad \lambda_{rem} = \lambda_{pre} \times \text{rem}$$

剩余时间越少，可再进球期望越低。这是结果随时间锁定的根本原因。

**② 红 / 黄牌修正**

$$\lambda^{self}_{rem} \mathrel{\*}= 0.72^{\,\text{自方红}} \times 1.08^{\,\text{对方红}} \times (1 - 0.012 \times \min(\text{自方黄}, 4))$$

- 每张**自方红牌**把自方剩余 λ 乘 `0.72`（`RED_SELF`）；
- 对手每张红牌让自方剩余 λ 乘 `1.08`（`RED_OPP`，多打一人略增）；
- **黄牌**为谨慎弱信号，每张压低 `1.2%`（`YELLOW_CAUTION`），最多计 4 张。

**③ 比分状态效应**

$$I = 0.28 \times (1 - \text{rem}), \qquad g = \frac{\min(|\text{净胜}|, 2)}{2}$$

领先方剩余 λ 乘 `(1 − I·g)`、落后方乘 `(1 + I·g)`。即**领先方趋于收缩护分、落后方压上进攻**，且强度 `I` 随比赛临近结束（rem→0）放大到最高 `0.28`（`STATE_MAX_INTENSITY`）。`g` 表示分差档位：1 球差=0.5，2 球及以上=1。

**④ 场面权重（实时技术统计）**

仅在传入 `stats`（双侧齐全）时生效。先用技术统计合成每侧**进攻压力指数** `P`：

$$P = w_{xg}\,xG + w_{sot}\,射正 + w_{shot}\,射门 + w_{cor}\,角球 + w_{poss}\,控球\%$$

权重 `xG` 信息量最高、控球率最低（见常数表）。得主队压力占比 `share = P_h / (P_h + P_a)`，
转成 λ 乘子并按**样本量缩放**（双方射门+角球总数越多越可信）：

$$\text{tilt} = (\text{share} - 0.5)\times 2, \quad \text{eff} = \text{tilt}\times \text{FIELD\_MAX}\times \min\!\Big(\tfrac{\text{样本}}{\text{FIELD\_SAMPLE\_FULL}}, 1\Big)$$

$$\lambda^h_{rem} \mathrel{*}= 1 + \text{eff}, \qquad \lambda^a_{rem} \mathrel{*}= 1 - \text{eff}$$

即**占优方剩余 λ 上调、劣势方下调**，平衡档最大 ±25%（`FIELD_MAX`）。
关键设计：**开场射门/角球少时样本缩放≈0，几乎不影响**；数据足（≥ `FIELD_SAMPLE_FULL`）才满强度，
避免开场零星一脚射门就大幅扰动赛前模型。`field.active/homeShare/lean/intensity` 全程透明暴露。

### 2.3 剩余进球分布与最终结果

对修正后的 `λ_rem(home/away)` 构建 8×8 进球矩阵，使用 **Dixon-Coles** 低分修正（沿用赛前引擎 `ACTIVE_PARAMS.rho`）：

$$P(i, j) = \text{Poisson}(i, \lambda^h_{rem}) \cdot \text{Poisson}(j, \lambda^a_{rem}) \cdot \tau_{i,j}(\rho)$$

归一化后逐格累加：**最终比分 = 当前比分 (gh, ga) + 剩余推演 (i, j)**，据此得到胜平负、Top 最终比分、以及"后续是否再进球"（取剩余 0 球概率的补）。最可能最终比分取与当前倾向 `lean` 同类中概率最高者，避免"看好主胜却给出平局比分"的割裂。

### 2.4 修正常数表

| 常数 | 值 | 含义 |
| --- | --- | --- |
| `REG_MINUTES` | 90 | 常规结束分钟（小组赛无加时） |
| `RED_SELF` | 0.72 | 每张自方红牌的剩余 λ 乘子 |
| `RED_OPP` | 1.08 | 每张对方红牌对自方的乘子 |
| `YELLOW_CAUTION` | 0.012 | 每张黄牌谨慎压低（≤4 张） |
| `STATE_MAX_INTENSITY` | 0.28 | 比分效应最大强度（rem→0 时达到） |
| `FIELD_MAX` | 0.25 | 场面权重最大乘子偏移（占优方最多 ×1.25 / 劣势方 ×0.75，平衡档） |
| `FIELD_SAMPLE_FULL` | 14 | 双方射门+角球达此数视为样本充分（强度满） |
| `PW_XG` / `PW_SOT` / `PW_SHOT` / `PW_COR` / `PW_POSS` | 1.0 / 0.45 / 0.12 / 0.08 / 0.015 | 进攻压力指数各项权重（xG 最高、控球率每 1% 计 0.015） |

> 常数为**保守标定、可解释优先**，非过拟合训练参数。
> 场面权重强度档位（平衡 ±25% / 保守 ±15% / 激进 ±40%）由 `FIELD_MAX` 决定，当前为平衡档。

## 3. 前端（`components/in-play-analysis.tsx`）

详情页 `/match/:id` 常驻挂载，**一张卡片自适配三种状态**：

| 阶段 | 徽标 | minute | 行为 |
| --- | --- | --- | --- |
| 未开赛 `upcoming` | 未开赛 | 0（rem 100%） | 开球前推演；默认折叠 |
| 进行中 `live` | LIVE（红色脉冲） | 实时比赛钟 | 每秒重算；默认展开 |
| 已完赛 `finished` | 已完赛 | 90（rem 0%） | 锁定最终结果；隐藏实时参数；默认折叠 |

- 标题统一为「赛中实时分析」，徽标与副标题随阶段切换。
- 实时分钟来自 `live-provider` 的 `useLiveInfo(matchKey)` + `liveClockFromStatus`/`tickClock`，仅进行中开启每秒 `tick`。
  - **秒针锚定固定的开球时间戳 `kickoffMs`**（确定性、跨刷新/跨设备一致，poller 重抓不归零）；分钟以 API 的 `elapsed` 为权威。详见 [数据拉取与实时机制 §三](./数据拉取与实时机制.md)。
- 红黄牌经 `useLiveEvents(matchKey)` 轮询事件统计；**技术统计经 `useLiveStats(matchKey, enabled, poll)`**：赛中每 25s 轮询 `/api/match-stats` 实时刷新、已完赛拉一次最终版、未开赛不获取。仅赛中且双侧齐全时才喂给引擎做场面权重。
- 展示：实时胜平负条（`ProbBar`）、两队再进球概率、最可能最终比分、剩余 λ（标签为「主/客队预计再进球」）、生效修正标签（红牌/护分/**场面偏向**）、**SofaScore 式技术统计对比条（全部八项：控球率/射门/射正/xG/角球/犯规/传球成功率/越位）**、一句中文研判与免责声明。

## 4. 约定

- **复用不重算**：赛中引擎只消费赛前 λ，不重新估计球队实力，保证与赛前预测同源一致。
- **纯函数无副作用**：`predictInPlay` 可单测、可在客户端逐秒运行，不产生后端负载。
- **透明可解释**：所有生效修正通过 `modifiers` 暴露并在 UI 标注，不做黑盒。
- **免责**：面向用户文案统一署名 opus4.8，明确"仅供娱乐，不构成投注建议"。
