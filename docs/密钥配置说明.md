# 密钥配置说明

本文说明如何创建和维护本地私有密钥文件：

`WORLD-CUP-AI-HUB/config/secrets/keys.local.mjs`

## 文件分工

`config/secrets/keys.example.mjs`

公开模板文件。可以提交到仓库，用来告诉别人需要哪些字段。这里不要填写真实 key。

`config/secrets/keys.local.mjs`

本地私有文件。真实 key 只填在这里。该文件已被 `.gitignore` 忽略，不应该提交到仓库。

`config/secrets/index.mjs`

统一读取入口。脚本通过它读取 key。读取优先级是：

1. 环境变量非空值
2. `keys.local.mjs` 非空值
3. `index.mjs` 内置默认值

## 新建 keys.local.mjs

如果本地没有 `keys.local.mjs`，在项目根目录运行：

```powershell
Copy-Item .\config\secrets\keys.example.mjs .\config\secrets\keys.local.mjs
```

然后打开：

```text
WORLD-CUP-AI-HUB/config/secrets/keys.local.mjs
```

把真实 key 填进去。

## 当前系统最少需要填写

实时数据主链路至少需要：

```js
API_FOOTBALL_KEY: "你的 API-Football key",
```

如果只查看静态页面或样例数据库，可以不配置任何真实 Key。`MIMO_API_KEY`、`GLM_API_KEY` 等字段只在运行对应可选脚本时需要。

## 字段说明

`API_FOOTBALL_KEY`

API-Football / API-SPORTS 的 key，来源是 `dashboard.api-football.com`。

用途：当前主力数据源，负责实时比分、比赛状态、阵容、事件、裁判、统计、伤停、球员名单等。

`MIMO_API_KEY`

小米 MiMo 的 key。

用途：联网搜索赛前信息，例如预测阵容、伤停、停赛、发布会、天气、赔率走势。

`FOOTBALL_DATA_API_KEY`

football-data.org 的 key。

用途：旧/备用比分状态链路 `scripts/ingest-scores.mjs`。如果不跑这条脚本，可以不填。

`GLM_API_KEY`

智谱 GLM 的 key。

用途：`scripts/ingest-factors.mjs` 生成“关键因素对比 / AI 分析”。如果不跑这条脚本，可以不填。

`MIMO_BASE_URL`、`MIMO_MODEL`、`MIMO_WEB_SEARCH`

MiMo 的接口地址、模型名、是否联网搜索。留空时会使用默认值。

`GLM_BASE_URL`、`GLM_MODEL`

GLM/OpenAI 兼容接口地址和模型名。留空时会使用默认值。

`API_FOOTBALL_DELAY_MS`、`API_FOOTBALL_RETRY_WAIT_MS`、`MIMO_SEARCH_DELAY_MS`

节流配置。正常情况下不用填，只有遇到限流或请求过快时再调整。

## 验证读取是否正常

运行下面命令，只会输出脱敏值：

```powershell
node --input-type=module -e "const { getApiKeys, maskKey } = await import('./config/secrets/index.mjs'); const k = await getApiKeys(); console.log({ API_FOOTBALL_KEY: maskKey(k.API_FOOTBALL_KEY), MIMO_API_KEY: maskKey(k.MIMO_API_KEY), MIMO_BASE_URL: maskKey(k.MIMO_BASE_URL), MIMO_MODEL: maskKey(k.MIMO_MODEL) })"
```

应该能看到 `API_FOOTBALL_KEY` 和 `MIMO_API_KEY` 是脱敏后的值。

## 验证没有泄露

运行：

```powershell
npm run audit:keys
```

期望结果：

```text
Key exposure audit passed
```

## 安全规则

不要把真实 key 写进：

- `keys.example.mjs`
- `.env.example`
- README 或文档
- 前端组件
- API response
- console 日志

真实 key 只允许放在：

- 本地 `config/secrets/keys.local.mjs`
- 部署环境的环境变量

## 开源前检查

开源或发给别人之前，确认：

```powershell
npm run audit:keys
```

确认 `.gitignore` 里包含：

```text
config/secrets/keys.local.mjs
```

仓库里应该保留 `keys.example.mjs`，不要提交 `keys.local.mjs`。
